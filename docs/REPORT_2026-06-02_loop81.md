# Loop 81 Report -- 2026-06-02

## Summary

**Closes the layout-duplication arc (executes Loop 80's Option A). `gen_rom.py`'s `pack_record()` no longer hardcodes the bit-shifts `<< 72`, `<< 68`, ...; it now DERIVES every field's position/width from the `.t27` SSOT via `tools/ssot_layout.py`. The 80-bit ROM layout now has exactly ONE functional source. Because the packer and the extractor now both read the same parser, the old consistency probe became partly by-construction -- so this loop adds an INDEPENDENT golden-offset oracle (`GOLDEN_LAYOUT`, hand-transcribed from rom_layout.t27 Section 2) whose sole job is to pin the parser. Critically verified: regenerating `format_rom.v` after the refactor is BYTE-IDENTICAL (sha 606877a0...), and a parser-drift mutation is now caught by the golden oracle (rc=1). Layout copies: 3 (Loop 78) -> 2 (Loop 80) -> 1 functional + 1 independent oracle (Loop 81).**

## The change

### `tools/gen_rom.py` -- packer derives from SSOT
- Added `import ssot_layout` (same directory; `sys.path` seeded from `__file__`).
- `pack_record()` now iterates `_FIELD_MAP` (logical arg -> SSOT FIELD_ name) and
  places each value at `ssot_layout.LAYOUT[name]`'s `(lo, width)`. The 256->0
  8-bit overflow warning and `phi_distance()` computation are unchanged.
- **Proof of behaviour preservation:** `python3 tools/gen_rom.py` regenerates
  `src/rtl/format_rom.v` byte-identically (`git status` clean, sha unchanged).
  The RTL ROM is therefore provably unaffected -- this is a pure source refactor.

### `test/test_ssot_layout_crosscheck.py` -- independent oracle
- Added `GOLDEN_LAYOUT`: an 11-entry `(hi, lo, width)` table transcribed by hand
  from `rom_layout.t27`. New check `parse_fields == GOLDEN_LAYOUT` runs first and
  prints per-field drift on mismatch.
- Rationale: with the packer now reading the parser, a parser bug (or an
  unintended edit to the FROZEN spec) would move packer + extractor together and
  could pass the consistency probe. The golden oracle is the independent witness
  that a verification check requires; updating it is the deliberate friction the
  "[Spec] FROZEN at Phase B" clause intends.
- The consistency probe is retained as defence-in-depth (catches a packer that
  ignores the layout / drops a field / lets fields bleed).
