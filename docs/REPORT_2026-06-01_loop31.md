## Critical Fix: LNS8 Antilog LUT

### Bug Description

The LNS8 decoder's 16-entry lookup table (`2^(frac/16) × 256`, rounded) had incorrect values at indices 3, 5, 6, and 7:

| Index | Wrong | Correct | Error | Root Cause |
|-------|-------|---------|-------|------------|
| 3     | 291   | 292     | −1    | floor instead of round |
| 5     | 317   | 318     | −1    | floor instead of round |
| 6     | 331   | 332     | −1    | floor instead of round |
| 7     | 345   | 347     | −2    | neither floor nor round |

### Why It Was Masked

The same wrong values appeared in all three reference sources:
- `src/rtl/lns8_decode.v` (RTL)
- `formal/fv_lns8.sv` (formal golden model)
- `test/test_decoders.py` (cocotb reference)

All three agreed, so every test passed despite the bug. The error was found by recomputing `round(256 × 2^(i/16))` from first principles.

### Impact

Maximum error of 2/256 ≈ 0.78% at index 7. For a conformance oracle this matters — downstream consumers trusting the decode output would accumulate the error.

### Fix

Three files corrected simultaneously:
- `src/rtl/lns8_decode.v` — entries 3→292, 5→318, 6→332, 7→347
- `formal/fv_lns8.sv` — golden LUT updated
- `test/test_decoders.py` — `ref_lns8()` LUT updated

## Research Findings

### Zero-Delay Gate-Level Simulation
Yosys can synthesize to generic cells (`synth -top ... -flatten`), then Icarus simulates with `simcells.v` providing behavioral models. This gives a zero-cost GLS flow without needing PDK timing models — useful for catching synthesis mismatches.

### SymbiYosys Cover Mode
Cover mode uses the SAT/SMT solver in reverse: instead of proving no counterexample exists, it finds input sequences that reach each `cover()` point. This proves reachability — a property BMC and k-induction cannot establish.
