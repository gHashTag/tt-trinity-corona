### 2. Bug Fix: BF16 NaN Test Vector

The initial BF16 NaN test had incorrect expected status bytes — byte[1] was 0xC0 instead of 0x00 (LSB-first byte ordering error). Fixed before first successful run.

### 3. Research Findings

**YosysHQ EQY:** Formal RTL-vs-netlist equivalence checker, usable but not production-certified. A 2025 Cornell paper warns EQY bugs can manifest as missed design bugs — use as supplement to GLS, not replacement.

**TTGF26a submission:** No standalone checklist document. Effective validation is passing GDS Action + Docs Action in GitHub. Submit via app.tinytapeout.com with `--gf` flag. Common failures: wrong PDK variable, DRC/LVS errors.

**GF180MCU timing:** 180nm node, 25-40 MHz achievable for decoder-class designs. OpenLane2 handles hold repair automatically.

**Coverage assessment:** 100% decoder GLS + formal BMC + exhaustive cocotb is stronger than typical TT submissions, which usually only run cocotb. Bounded formal proofs are considered acceptable for sign-off when bounds exceed design state depth (DVCon 2025 paper).

**MXDOTP/Vmxdotp:** RISC-V ISA extensions for OCP MX dot products define FP9 (E5M3) as intermediate accumulation format — potential future catalog addition.
