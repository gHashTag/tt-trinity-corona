## Critical PDN Fix

### Problem Chain

1. LibreLane default: DESIGN_IS_CORE=true -> generates full PDN with Metal5 straps
2. TT precheck: Metal5 strictly forbidden for user tiles (reserved for TT integration)
3. Loop 50 fix: `FP_PDN_MULTILAYER: false` -> OpenLane 1 variable, **silently ignored** by LibreLane

### Solution

`DESIGN_IS_CORE: false` in config.json tells LibreLane this is a macro, not a standalone core. In macro mode, LibreLane generates only lower-metal PDN (Metal1 stdcell rails) and relies on the parent design (TT integration) for top-level power distribution via Metal5.
