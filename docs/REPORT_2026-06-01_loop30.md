### 3. Audit: BF16 Byte-Ordering False Alarm

The audit agent claimed BF16 bytes were reversed. Investigation proved this
is incorrect. The shift register `{ui_in, data_in_buf[31:8]}` implements
LSB-first accumulation:
- Send [0x00, 0x3F] → data_in_buf[31:16] = 0x3F00 (correct BF16 for 0.5)
- All 7 BF16 key-value tests pass with correct byte ordering

## Research Findings

### TT Tile Budget
4×4 tiles is comfortable for 2,306 cells. Routing overhead (tap cells,
antenna diodes, buffers) typically doubles naive area. Real TT projects
report routing as the primary constraint, not cell count. 16 tiles provides
generous margin for timing closure at 50 MHz on GF180MCU 180nm.

### k-Induction vs IC3
k-Induction cannot discover auxiliary invariants automatically — that's why
we needed AI1-AI7 manually. IC3/PDR can auto-discover invariants and is
strictly more powerful for some properties. Running both is standard practice.
rIC3 (top at HWMCC'24) can serve as a SymbiYosys backend for cross-checking.

### Cocotb Gate-Level Simulation
Same cocotb testbench works for zero-delay GLS on GF180MCU. Icarus has
limited SDF annotation support, so timing-annotated simulation needs a
commercial tool. wafer-space/gf180mcu-project-template demonstrates this flow.

### Workshop Targets
WOSET (Open-Source EDA) is the best venue — last ran Nov 2024, no 2026
edition announced yet. 4-page papers + 1 page refs. CAV 2026 deadline was
Jan 2026 (passed). Monitor woset-workshop.github.io for next CFP.
