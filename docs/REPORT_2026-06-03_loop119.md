## Why

The prior loops found, characterized, and committed a verification harness for a
systematic bug: 8/9 GoldenFloat `gfN_add`/`gfN_mul` units were broken (only the
`[Verified]` gf16 was correct). The pilot fix (GF8, the only rung where a full
65536-pair exhaustive sweep is feasible) was the proposed next step. It is safe to
do autonomously because: the fix is fully exhaustively verifiable, GF8 was already
broken (a fix cannot regress it), and it is a source change -- the Gamma tapeout
(submitted 2026-05-17) is frozen, so silicon is unaffected.

## Implementation (in tt-trinity-gamma)

- `src/gf8_mul.v`: full-width mantissa product; `prod[9] -> exp+1` else stay;
  mantissa = bits below the leading 1; guard/round/sticky; round-to-nearest
  ties-to-zero (gf16_mul convention). Removed the truncating `[8:0]` reg and the
  wrong exp bump / mantissa slice.
- `src/gf8_add.v`: significands `{1, mant, 3'b000}`; alignment captures a sticky
  bit; leading 1 found by priority and renormalized to a fixed position; 4-bit
  mantissa rounded to nearest, ties-to-even; e0m0 "0.125 hole" and round-to-Inf
  at the top binade handled. Dropped an unused `BIAS` localparam (verilator clean).
- `test/gf8_exhaustive.py`: exhaustive 65536-pair cross-check vs a value-based
  reference (decode -> exact float -> op -> compare in ULPs). Non-circular -- it
  checks the produced value, not the RTL's bit slicing. Exit 0 iff all pass.
- `docs/GF_ARITH_FINDINGS.md`: updated -- GF8 marked FIXED, fix documented, the
  remaining-rung plan now points at GF8 as the template for the small rungs.

## Research (this loop, web available)

The episode is a textbook narrow-format rounding lesson: a guard/round/sticky path
is not optional when the stored mantissa is only 4 bits. The wider gf16 unit
"gets away with" a truncating alignment because 9 mantissa bits make the dropped
low bits negligible; at 4 bits the same shortcut produces multi-ULP error on
subtractive cancellation (Goldberg, "What Every Computer Scientist Should Know
About Floating-Point Arithmetic"; Muller et al., *Handbook of Floating-Point
Arithmetic*, on why GRS bits are required for faithful rounding). The verification
mirrors the chip's own philosophy -- an exhaustive sweep against an independent
reference, the same evidence layer Corona uses for its decoders -- and is the
cleanest possible check for an 8-bit format (the entire input space fits).
