## Critical Fix: MXFP8 E4M3 Subnormal Exponent

### Bug Description

OCP MX FP8 E4M3 has bias=7, so E_min = 1 - 7 = -6. The code incorrectly used -7 as the base exponent (confusing bias with E_min), making all three subnormal FP32 exponents +1 too high:

| Subnormal | Wrong | Correct | Derivation |
|-----------|-------|---------|------------|
| mant[2]=1 | 121 | 120 | -6 + (-1) + 127 = 120 |
| mant[1]=1 | 120 | 119 | -6 + (-2) + 127 = 119 |
| mant=001 | 119 | 118 | -6 + (-3) + 127 = 118 |

### Impact

Every subnormal value decoded to exactly 2x its correct magnitude. The largest subnormal (0x07 = exp=0, mant=111) decoded to 1.75 instead of 0.013671875 — a 128x error that creates a massive discontinuity at the subnormal/normal boundary. The smallest normal (0x08 = exp=1, mant=000) correctly decodes to 0.015625, so the largest subnormal (1.75) was *larger* than the smallest normal.

### Root Cause

The RTL comment said `-7+1+127 = 121` but the correct derivation is `-6-1+127 = 120`. The error is `E_min = -bias` vs the correct `E_min = 1 - bias`. Same error propagated to `fv_mxfp8_e4m3.sv` and `test_decoders.py`.

### Cross-Reference: FNUZ is Correct

The E4M3 FNUZ decoder (bias=8) uses exponents 119/118/117, which are correct for E_min = 1-8 = -7. The MX E4M3 values should be exactly 1 higher (120/119/118) since its bias is 1 less.

### Files Fixed

- `src/rtl/mxfp8_e4m3_decode.v` — lines 40, 43, 46: 121→120, 120→119, 119→118
- `formal/fv_mxfp8_e4m3.sv` — lines 30, 32, 34: same corrections
- `test/test_decoders.py` — `ref_mxfp8_e4m3()` lines 129, 131, 135: same corrections
