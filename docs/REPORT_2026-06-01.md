## 2. Scientific Literature Review

### Key papers by topic (relevance to Corona)

**Posit hardware:**
- PPU (Rossi+ 2023): 7-15% area overhead for posit8/16 in RISC-V. Validates posit8 Tier-1 (300-800 cells).
- SPADE (2025): SIMD posit at TSMC 28nm, 1.38 GHz. Confirms posit32 feasibility but high cell count.

**Takum (standing counterexample to FL-002):**
- Hunhold 2024a (arXiv:2404.18603): Takum beats posit at dynamic range. Key format for the catalog.
- Hunhold 2024b (arXiv:2408.10594): VHDL codec, 38% latency reduction vs posit. **License unclear -- R2 risk.**
- Hunhold 2024c (arXiv:2412.20273): Integer representation analysis. Mathematical reference for ROM entries.
- Hunhold 2024d (arXiv:2412.20268): Exceptional stability in sparse solvers. Strengthens counterexample.

**BFloat16/TF32:**
- Trivial decode (50-200 cells). bf16 = truncated FP32; tf32 = 19-bit FP32 mode.
- Open-source RTL exists (Essenceia/BFloat16 on GitHub). Safe early-win for Phase C.

**MX (Microscaling):**
- OCP MX v1.0 spec (AMD/Arm/Intel/Meta/Microsoft/NVIDIA/Qualcomm consortium).
- MXFP8 element decode is simple (case statement). Block structure with shared scale is out of scope for Corona (per-element decode only).

**LNS (Logarithmic Number System):**
- LNS-Madam (NVIDIA 2022): 90% energy reduction vs FP32 for 8-bit LNS training. Validates lns8 Tier-1.
- Logarithmic-Posit (DAC 2024): LNS+posit hybrid. Not in catalog but candidate for future SSOT expansion.

**GF180MCU PDK:**
- 180nm, ~52 kGates/mm^2 (vs ~160 kGates/mm^2 for SKY130). No open SRAM macro.
- 2.1x density penalty confirmed by community estimates. ROM must be combinational (case-statement synthesis).

**GoldenFloat (GF4-GF256):**
- **No published academic work found.** GoldenFloat appears original to TRI-NET.
- Nearest precedent: Bergman (1957) golden ratio base; Stakhov (1980-2021) Fibonacci computer arithmetic.
- PLAN.md correctly classifies as [Verified] for algebra, [Open conjecture] for moat (FL-002).

**ROM synthesis:**
- Combinational ROM (case statement) exploits data redundancy via Boolean optimization.
- 77 records x 80 bits estimated at 2,000-6,000 cells depending on data structure.
- 4x4 tiles (~8,000 cells) is tight but feasible for ROM + 5-8 Tier-1 modules.

**Why this first:** If GF180MCU synthesis fails or density is worse than 2.1x,
everything downstream changes. Better to know now.

### Option B: "ROM Generator" -- Build the Verilog ROM Emitter

Focus on implementing the Verilog ROM emitter as the 17th output language of
gen_formats_catalog.py. This is the Phase B critical path item. Deliverable:
generated format_rom.v with all 77 records, cocotb readback test passing.

**Why this first:** The ROM is the core of the chip. Without it, Tier-1 module
work is speculative. The emitter also validates the SSOT -> silicon chain.

### Option C: "MVP-5 Blitz" -- Implement 5 Tier-1 Decoders

Skip ahead to implementing the 5 simplest Tier-1 decode modules (bf16, mxfp8,
lns8, bcd, posit8) directly in the stub, with round-trip tests. This builds
the most RTL in the shortest time and de-risks Phase C.

**Why this first:** bf16 and mxfp8 are ~50-200 cells each and can be written
in an hour. Having 5 working decoders makes the MVP tangible and testable,
even before the ROM emitter is done.
