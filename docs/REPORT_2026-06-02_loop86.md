## Research (this loop, web available)

NF4 is information-theoretically motivated, not a float format: 16 quantiles of
N(0,1) with equal probability mass, generated by
`create_normal_map(offset=0.9677083, use_extra_value=True)` and hardcoded in
bitsandbytes to avoid a runtime scipy dependency. The `offset` bounds the tail
quantiles (else +/-inf); `use_extra_value=True` yields the asymmetric **7
negative / exact 0.0 / 8 positive** layout (zero is exact so sparse weights
quantize losslessly), normalized to [-1, 1]. Corona's NF4 LUT reproduces these
exact float32 constants; FP4 E2M1 is the OCP MX element type and is fully
algorithmic.

Sources: [QLoRA (arXiv:2305.14314)](https://arxiv.org/pdf/2305.14314),
[bitsandbytes functional.py](https://github.com/bitsandbytes-foundation/bitsandbytes/blob/main/bitsandbytes/functional.py),
[NF4 asymmetry analysis (McCormick)](https://mccormickml.com/2024/09/14/qlora-and-4bit-quantization/).
