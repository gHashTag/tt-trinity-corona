## Research: Scientific Literature Survey

### FP8 E4M3 FNUZ — Production Traction
AMD MI300/MI325X (CDNA3) uses FNUZ natively. vLLM implements
`normalize_e4m3fn_to_e4m3fnuz` for cross-vendor inference serving. The format
differs from OCP E4M3 by exactly bias (7→8) and NaN encoding (0x80 repurposed).
This decoder demonstrates Corona can distinguish the two at element level.

### OCP MX — Still v1.0
No v1.1 or v2.0 published. Vendor extensions emerging: NVFP4 (Nvidia, block-16),
HiFloat4 (Huawei, 3-level hierarchy), MX-SAFE (adaptive E/M). All differ at
block level, not element level — Corona's element decoders remain valid.

### Sub-8-bit Landscape — Fragmented, Not Converging
MXFP4, NVFP4, HiFloat4 all share E2M1 element format but differ in blocking.
FP6 E3M2/E2M3 hardware-supported on Blackwell. No new element-level formats
beyond what Corona already covers.

### IEEE P3109 (Posit) — Still In Draft
Formal verification presented at ARITH 2025. Exponent bias decisions still
being refined. Corona's Posit8 decoder remains well-timed.

### GF180MCU Ecosystem — Active
wafer.space GF180MCU Run 1 delivered dies March 2026; Run 2 announced May 2026.
TTGF26a (283 designs) expects chips October 2026. Pipeline validated.
