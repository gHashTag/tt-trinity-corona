**Finding #2 detail**: The FSM accepts any byte_count 0-15. Decoders expect
specific lengths (BF16=2, TF32=3, FP4=1, etc.). Adding per-format validation
would require a 77-entry lookup table (~200 cells) and is unnecessary: the
host protocol stack is responsible for sending correct byte_count. This is
consistent with the minimalist silicon approach (no runtime error handling,
conformance oracle only).

## Research: Key Discoveries

### IEEE P3109 Formal Verification (ARITH 2025, FLoPS)
Two papers formally verify the IEEE P3109 `decode` function — exactly what
Corona's Tier-1 decoders implement. P3109 defines parametric ML floating-point
formats (<16 bits). Wintersteiger et al. used Imandra theorem prover; FLoPS
used Lean 4. Both prove **totality**: every input produces a valid output.
This is the same property Corona's SymbiYosys proofs verify.

**Relevance**: Corona's FP4, FP6, FP8, BF16 decoders are P3109-adjacent.
The formal methodology aligns with Corona's approach (BMC exhaustive proof).

### NVFP4 — Dominant New 4-bit Format
NVIDIA Blackwell introduces NVFP4: E2M1 elements + E4M3 block scale (per 16)
+ FP32 global scale. Three-tier scaling hierarchy. Different from MXFP4
(E8M0 scale per 32). This is now the production 4-bit format for LLM inference.

**Impact**: NVFP4 is the highest-priority candidate for future SSOT addition
(format_index 78+). ROM has capacity: 7-bit index supports 128 entries,
currently 80 used.
