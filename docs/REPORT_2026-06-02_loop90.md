## Implementation

- Definitional refs: `bf16 = (bits<<16)`; `tf32 = sign|exp|mant repacked into fp32
  fields` (NaN/Inf payloads pass through, matching the truncation definition).
- Value-based refs: compute the exact real value from (sign, exp-bias, mantissa,
  subnormal) and let `struct` produce float32 bits; specials per spec

  fp6 e3m2/e2m3 no Inf/NaN).

  output compared. Skips gracefully (rc=0) without iverilog.

## Research (this loop, web available)

The value-based method exploits a key property: every code in these sub-16-bit
floats represents a real number that is EXACTLY representable in IEEE-754
float32 (their exponent range and precision both fit inside fp32), so
`float32(exact_value)` is the unique correct decode -- no rounding ambiguity.
This makes "compute the value, emit fp32" a rigorous oracle, independent of the
RTL's bias-subtract + mantissa-shift construction. The three FP8 special-value
regimes were confirmed against the standards:
- **e5m2** -- IEEE-style: Inf at exp=all-ones,m=0; NaN m!=0 (OCP OFP8).
- **e4m3 (OCP)** -- no Inf; NaN only at S.1111.111 (Micikevicius et al.,
  arXiv:2209.05433).
- **e4m3 FNUZ** -- AMD/Graphcore: bias 8, 0x80 the sole NaN, no negative zero.

Sources: [FP8 formats (arXiv:2209.05433)](https://arxiv.org/abs/2209.05433),
[OCP OFP8 Rev 1.0](https://www.opencompute.org/documents/ocp-8-bit-floating-point-specification-ofp8-revision-1-0-2023-12-01-pdf-1),
[bfloat16 / TF32 (NVIDIA)](https://developer.nvidia.com/blog/accelerating-ai-training-with-tf32-tensor-cores/).
