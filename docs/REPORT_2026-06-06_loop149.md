implements BIND as ternary multiply (81/81 vs the reference); frozen source
untouched. Verified by `test/leaf_audit.py`, green on both dies.

## Why

After the arithmetic, crypto/identity, and fabric layers were audited, the leaf
blocks (host bus / PLL / status / ALU) were the only instantiated logic never
functionally checked. Auditing them lets the program state the silicon is fully
characterized, not just "the big blocks are." Each has a checkable contract (a
fractional-divider rate, the WB read/write protocol, a documented bit layout, and
a {-1,0,+1} ternary algebra).

## Research (this loop)

In MAP-style Vector Symbolic Architectures over a bipolar/ternary alphabet, the
three core operators are bundle (elementwise add, superposition), bind (elementwise
multiply, association), and permute. Bind must be invertible and must satisfy the
absorbing/identity laws of multiplication -- bind(x, +1) = x and bind(x, 0) = 0 --
which is exactly what makes unbinding (bind again with the same key) recover the
operand. The shipped op7 `sa ^ sb` (XOR of two's-complement signed values)
satisfies none of these for the zero element and self-bind, so it is not a valid
bind; the header's "(XOR)" was the bipolar {-1,+1} shorthand (where bind = multiply
= XOR of the sign/encoding bits) applied to the wrong representation. v2 restores
multiply. As with the rest of this audit, the bug is RTL diverging from the
contract it states, and -- like bitnet and multi_tile_receipt -- a second design
choice (driving the unit with random bits into a liveness ring) makes it benign on
the actual die, which is why grading needs the wiring, not just the truth table.
