## Scientific Research

### IEEE P3109 v3.2.1 -- Arithmetic Formats for ML

Key findings from the Interim Report (still draft, not for conformance):
- **7 binary8 formats**: binary8p1 through binary8p7 (P = significand width including implicit bit)
- **Bias reversal (July 7, 2025)**: For P>1, bias = 2^(W-1) instead of 2^(W-1)-1. Material change from IEEE 754 convention.
- **No signed zero**: -0 encoding repurposed as NaN
- **Supernormals**: Infinity shares max exponent with finite values
- **FLoPS errata**: (1) emax formula incorrect for P=1 and P=2; (2) encode function error for unsigned extended infinity
- **Action**: Cross-reference .t27 catalog with P3109 binary8 formats post-submission. Not urgent for TTGF26a deadline.

### MX Ecosystem Update

Base element encodings (MXFP8/MXFP6/MXFP4/MXINT8) unchanged from OCP v1.0. New block-level research:
- **MX+** (MICRO 2025): Repurposes outlier exponent bits as extended mantissa (+42% MXFP4 accuracy)
- **MXDOTP** (ASAP 2025): First RISC-V ISA extension for MX dot products (Snitch core, 356 GFLOPS/W)
- **Vmxdotp** (DATE 2026): Extends MXDOTP to RVV 1.0, MXFP8+MXFP4 inputs, SwissChips/Huawei funded
- **Impact on Corona**: None — element-level encodings in the ROM are stable. MX hardware ecosystem validates Corona's format catalog relevance.

### GF180MCU Reliability

No published silicon reliability data (SER, alpha sensitivity, aging) for GF180MCU 180nm from any source. 180nm has inherent advantage over sub-100nm nodes (higher critical charge). PDK DRM covers operating conditions (-40C to 125C) but not radiation effects. Acceptable for lab/educational use.

### Post-Silicon Validation Resources

- **Spacely** (Fermilab, arXiv 2406.15181): Open-source Python post-silicon framework for academic ASICs
- **RP2350 demoboard**: Standard TT bring-up platform, 8+8+8 pins, MicroPython scripting
- **Chips expected**: ~2026-10-01, delivery ~2026-11-15
- **Efabless context**: Shut down Feb 2025, ~500 TT designs stranded. Recovery via wafer.space (Tim Ansell).

### Novelty Confirmation (Strengthened)

Corona is the **first silicon-level format oracle** — the novelty of embedding a conformance reference in ROM on actual silicon as a measurable physical artifact is unique.
