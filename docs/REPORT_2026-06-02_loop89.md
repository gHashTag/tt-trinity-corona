## Research (this loop, web available)

The six decoders are standard, well-specified primitives; the relevant point is
that each has a *canonical closed form* that serves as an oracle independent of
any RTL or model:
- **int4/int8** -- two's-complement sign extension (the universal signed-integer
  convention).
- **bcd** -- packed BCD: each nibble a decimal digit, value = `10*tens + ones`;
  the `valid` flag enforces the digit<=9 invariant that distinguishes BCD from a
  raw byte.
- **bitnet** -- BitNet b1.58 ternary weights {-1, 0, +1} (Ma et al., 2024,
  arXiv:2402.17764); the 4th code (0b11) is reserved -> NaN here.
- **e8m0** -- OCP MX shared-scale exponent (2^(e-127), 0xFF = NaN per MX v1.0).
- **mxint8** -- OCP MX 8-bit integer element with an implicit 2^-6 scale.

Because each value is exactly representable (or an exact power of two / k*2^-6),
the closed form matches the RTL bit-for-bit with no rounding ambiguity.

Sources: [BitNet b1.58 (arXiv:2402.17764)](https://arxiv.org/abs/2402.17764),
[OCP MX v1.0](https://www.opencompute.org/documents/ocp-microscaling-formats-mx-v1-0-spec-final-pdf),
[two's complement](https://en.wikipedia.org/wiki/Two%27s_complement).
