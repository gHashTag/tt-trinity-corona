## Research (this loop, web available)

The episode is a clean example of the meta-test catching a measurement bug in
itself: a guard that computes a metric from the filesystem must avoid counting its
own source (string-literal self-reference), and a discovery rule keyed on a bare
substring is fragile when test files legitimately mention the very tokens used to
classify them. The robust fix -- match a real module-level import, not the
substring -- is the standard "parse structure, not text" lesson. The guard now
closes the recurring prose-count drift class (found in Loops 78, 115, 116) the
same way the gate count became self-correcting in Loop 115.

Sources: [test discovery by structure, not substring](https://docs.pytest.org/en/stable/explanation/goodpractices.html#test-discovery),
[single source of truth / docs drift](https://dev.to/codinsonn/the-joy-of-single-sources-of-truth-277o).
