## Critical Fix 1: Power Pin Trimming (FP_PDN_SKIPTRIM)

### Root Cause Chain

1. Design utilization: 2,308 cells / ~16,000 budget = **14%**
2. OpenROAD PDN: generates Metal4 power stripes, then **trims** them to placed cell bounds
3. Low utilization -> cells concentrated in center -> stripes trimmed 15.38 um from die edges
4. TT precheck: VGND/VDPWR must be within **10 um** of top and bottom die edges
5. Result: 40 pin check errors (20 VGND + 20 VDPWR), each failing both top and bottom

### Solution

`FP_PDN_SKIPTRIM: true` passes `-skip_trim` to OpenROAD's `pdngen`, preventing stripe trimming. Power stripes extend to full die area regardless of cell placement. This is the same approach used by the wafer-space/gf180mcu-project-template.

## Critical Fix 2: Cocotb 1.8 Compatibility

### Root Cause

TT's `gl_test` action installs `cocotb==1.8.0`. Our tests used `BinaryValue.to_unsigned()` (added in cocotb 2.0). All 44 GL tests crashed immediately with:

```
AttributeError: 'BinaryValue' object has no attribute 'to_unsigned'
```

### Solution

Replaced all 14 occurrences of `.value.to_unsigned()` with `.value.integer` across 3 test files. `.value.integer` works in both cocotb 1.8 and 2.0 for unsigned signals.

| File | Occurrences Fixed |
|------|-------------------|
| test/test_anchor.py | 8 |
| test/test_decoders.py | 2 |
| test/test_stress.py | 2 |
| **Total** | **14** (includes helpers called by 44 tests) |
