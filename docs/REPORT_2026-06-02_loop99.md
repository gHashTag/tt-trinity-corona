## Research (this loop, web available)

The technique rests on a standard simulator capability: Verilog `force` overrides
a net's value procedurally, letting a testbench drive a signal that has no
structural driver -- here the formal harness's `(* anyconst *)` input, which the
SMT engine treats as a free symbolic value but a simulator leaves undriven.
Sweeping the forced input and reading the golden hierarchically turns a
formal-only construct into something exhaustively *simulatable* for the finite
decoder input space, so the golden can be validated against an external reference
without invoking the model checker. (`force`/`release` are defined in IEEE 1800;
hierarchical references let the TB observe internal `golden` without adding
ports.)

Sources: [Verilog force/release (IEEE 1800 / sutherland-hdl overview)](https://www.verilog.com/VerilogBNF.html),
[SystemVerilog hierarchical references (ChipVerify)](https://www.chipverify.com/systemverilog/systemverilog-hierarchical-reference).
