## Precheck Root Cause Analysis

### What happened

| Job | Result |
|-----|--------|

| GL tests | **44/44 PASS** (cocotb .value.integer fix works) |

| Viewer | FAIL (cosmetic, expected on forks) |

### Why FP_PDN_SKIPTRIM didn't help

`FP_PDN_SKIPTRIM=true` prevents OpenROAD from trimming unconnected metal stubs. But the real problem was never about trimming — the Metal4 horizontal stripes were never placed close enough to the die edges in the first place.

Default GF180MCU PDN parameters:
- `HOFFSET ≈ 16.65 um` — first horizontal stripe already beyond 10 um limit
- `HPITCH ≈ 153.18 um` — wide spacing, few stripes

With die height 736.96 um and default offset/pitch:
- Bottom stripe: y = 16.65 um (FAILS: > 10 um)
- Top stripe: y ≈ 629.37 um → 107.59 um from top (FAILS)

### The fix

```json
"FP_PDN_HOFFSET": 5,
"FP_PDN_HPITCH": 75,
"FP_PDN_VOFFSET": 5,
"FP_PDN_VPITCH": 75
```

New stripe positions: y = 5, 80, 155, 230, 305, 380, 455, 530, 605, 680, 730...
- Bottom: 5 um from edge (PASSES: < 10 um)
- Top: 736.96 - 730 = 6.96 um from edge (PASSES: < 10 um)
