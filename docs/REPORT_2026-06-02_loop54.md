## Precheck Root Cause Analysis

### Three fix iterations

**Iteration 1: FP_PDN_SKIPTRIM=true** — No effect. Prevents stub trimming, but stripes were never placed near edges.

**Iteration 2: FP_PDN_HOFFSET=5, FP_PDN_HPITCH=75** — No effect. LibreLane 3.x dropped the `FP_` prefix — all `FP_PDN_*` variables are silently ignored. Distance still 15.38 um. Errors increased from 40 to 76 (more vertical stripes, each still bounded by core area).

**Iteration 3: BOTTOM_MARGIN_MULT=1, TOP_MARGIN_MULT=1** — **FIXED.** The real control is `BOTTOM_MARGIN_MULT` / `TOP_MARGIN_MULT` which set the core area margin in multiples of the standard cell site height (3.92 um for GF180MCU MCU7T5V0). Default=4 gives 4×3.92=15.68 um margin (matching measured 15.38 um). Setting to 1 gives 3.92 um (< 10 um limit). This matches TT's own multiplexer template generator (`tt_block/build.py`).
