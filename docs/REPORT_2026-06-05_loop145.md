## Research (this loop, web available)

Posits (Gustafson) earn their accuracy from the variable-length regime: codes near
1.0 spend their bits on fraction (high precision), codes far from 1.0 spend them on
the regime run (wide dynamic range), and the 2's-complement code order is monotonic in
value -- which is exactly the property the encoder verification exploits (the nearest
posit is one of the +-1 integer neighbours, so "is this the nearest?" is a two-compare
check, no separate reference encoder needed). The old stub's fixed-field layout threw
all of that away. The decoder is exhaustively checkable (16-bit input) against the
exact rational value; together they pin the codec bit-for-bit where it matters and to
<= 1 ULP where rounding is involved.
