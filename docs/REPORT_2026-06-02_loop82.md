## The gap

Three representations of the same packed ROM, with the generator as the intended source of record:

```
gen_rom.py CATALOG+pack_record  --emit-->  format_rom.v (committed RTL, ships)
                                              |
                                  make rom-golden checks RTL == tb_rom_golden.v
                                              |
                              tb_rom_golden.v golden_lo/mid/hi (hand-coded)
```

`format_rom.v` carries "AUTO-GENERATED ... DO NOT EDIT BY HAND", yet it is committed, not built -- and CI never regenerated-and-diffed it. The Loop 79-81 cross-checks pin the *packer* to the SSOT layout, but not the *emitted file* to the packer. So the failure mode "edit CATALOG, forget `python3 tools/gen_rom.py`" was undetected.

## Implementation

`test/test_rom_emitted_golden.py` (stdlib only, <1s):

- **Check A -- freshness:** emit to a temp dir via `gen_rom.generate_verilog(CATALOG, tmp)`, read the committed `format_rom.v`, assert byte-identical. On failure prints the first differing line and `run python3 tools/gen_rom.py`. (Working tree is never mutated -- safer than `git diff --exit-code`.)
- **Check B -- GLS golden vs generator:** regex-parse `golden_lo/mid/hi[idx]` from `tb_rom_golden.v`, reconstruct each 80-bit word `(hi<<64)|(mid<<32)|lo`, assert it equals `pack_record(CATALOG[fmt_id])` for all 80.
- CI: new step `ROM freshness + GLS-golden vs generator` after the SSOT cross-check.

## Research (this loop, web available)

The "regenerate-and-diff" CI gate is the established pattern for committed generated artifacts (golden/snapshot testing applied to codegen): regenerate in CI, fail on any difference, print the diff, require generator determinism. This test is the in-Python form of that gate, and adds a second assertion (B) tying the hand-coded GLS oracle to the generator -- something a plain `git diff` would miss because `tb_rom_golden.v` is not a generated file.

| Practice | This loop |
| --- | --- |
| `make generate && git diff --exit-code` | Check A, but in-process and without mutating the tree |
| print diff on failure | first-differing-line message + remediation hint |
| generator must be deterministic | confirmed Loop 81 (byte-identical regen, fixed sha) |
| golden/snapshot equality | extended to the hand-coded GLS golden (Check B) |

Sources: [git-diff docs](https://git-scm.com/docs/git-diff), [pre-commit quality gates](https://dev.to/cwprogram/implementing-quality-checks-in-your-git-workflow-with-hooks-and-pre-commit-4iip).
