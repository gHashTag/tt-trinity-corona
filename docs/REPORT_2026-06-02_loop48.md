## GDS Diagnosis and Fix

### Root Cause: Die-Area Auto-Sizing

GDS runs #26771645826 and #26771717056 both failed with:
```
[GRT-0209] Pin clk is completely outside the die area and cannot be routed.
Design area: 0.0 0.0 249.755 267.675
Template DEF area: 0.0 0.0 1440.32 736.96
```

LibreLane 3.0.3 auto-sized the die to ~250x268um based on cell count (2,308 cells), instead of using the 4x4 tile area (1440x737um) from the DEF template. The clk pin at y=736.46um was outside the auto-sized area.

**Fix v1:** Added `FP_SIZING: absolute` and `DIE_AREA` to `src/config.json`.
**Result:** GDS hardening PASSED, but precheck failed: Metal5 on power pins (VGND/VDPWR), 70 LEF errors. LibreLane used Metal5 for PDN even with RT_MAX_LAYER=Metal4.

**Fix v2:** Added `RT_MAX_LAYER: Metal4` and `FP_PDN_UPPER_LAYER: Metal4` to `config.json`. Also fixed cocotb `unit=` to `units=` (cocotb 2.0 API) for gl_test compatibility. GDS run #26773762781 in progress.
