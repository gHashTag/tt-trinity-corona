## Research (this loop)

The phi_d2d_lite bug is a classic asynchronous-serial start-bit detection error.
A UART-style receiver must sample the start bit ONCE to mark the bit boundary and
then sample each subsequent bit exactly once; robust receivers oversample (e.g. 16x)
and re-confirm the start at the MIDPOINT of the same bit, never at the next bit
boundary. This RX instead re-confirms one full bit-period later (RX_DETECT_START),
which lands on the first data bit -- so a 1-cycle start (1x sampling, which is what
the matching TX produces) is structurally undecodable. The lesson mirrors the rest
of the audit: a transmitter and a receiver are two halves of one contract, and each
must be tested against the other (or against the wire format), not in isolation --
the shipped TX and RX were each "reasonable" but mutually incompatible, which only a
loopback (or a wire-format reference) exposes.
