## Research (this loop, web available)

GitHub Actions job summaries (`$GITHUB_STEP_SUMMARY`) are the native mechanism for
rendering a step's results as markdown in the run UI -- the lightweight
alternative to uploading a JUnit artifact when the goal is human-readable
at-a-glance status. Writing the gate table there follows the convention of
surfacing test results where reviewers look, without adding a parsing/upload
dependency. The write is best-effort so observability never gates correctness.

Sources: [GitHub Actions job summaries](https://docs.github.com/en/actions/using-workflows/workflow-commands-for-github-actions#adding-a-job-summary),
[Actions environment files](https://docs.github.com/en/actions/learn-github-actions/variables).
