## Research (this loop, web available)

Post-silicon validation literature endorses exactly this approach: rather than
hand-authoring expected values, run identical stimulus through the model and use
its outputs as the golden reference, so simulation and silicon are compared on a
common truth. The key rigor point, stated explicitly in the literature: golden
vectors derived from RTL "encode the bug" if the RTL is faulty, so the
highest-confidence setup pairs RTL-derived vectors with an INDEPENDENT reference
model that cross-checks them. Corona now has precisely that chain: spec-derived
independent references (Loops 86-90) <-> actual RTL (iverilog) <-> bring-up oracle
(`test_corona.py`), each cross-checked. Loop 92 already demonstrated the payoff
by catching 5 hand-written E5M2 errors; this loop closes the remaining oracle
surfaces (aliases, coverage, anchor, cardinality).

Sources: [Post-silicon validation (Grokipedia)](https://grokipedia.com/page/Post-silicon_validation),

[Post-silicon validation: opportunities & challenges (DAC'10)](https://people.eecs.berkeley.edu/~sseshia/pubdir/postSi-dac10.pdf).
