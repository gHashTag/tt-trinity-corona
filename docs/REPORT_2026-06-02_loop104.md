## Research (this loop, web available)

Architecture Decision Records (Michael Nygard, 2011) are the standard lightweight
practice for capturing a decision, its context, and consequences at the time it is
made, so the "why" survives after the "what" is in the code. The well-known
failure mode this loop fixes is the *retroactive gap*: decisions get made under
deadline and the ADRs are skipped, leaving the rationale only in people's heads (or
lost). Recording them now -- accurately, from the implemented artifact -- restores
the decision trail the project's own process requires, and a Q->ADR map keeps the
question list from reading as stale open work.

Sources: [Documenting Architecture Decisions (Nygard)](https://cognitect.com/blog/2011/11/15/documenting-architecture-decisions),
[ADR GitHub organization / examples](https://adr.github.io/).
