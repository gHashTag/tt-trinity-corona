## Implementation

### 1. P14 Formal Assertion (STATUS Exit Correctness)

Added to `src/rtl/tt_um_trinity_corona.v`:
```systemverilog
// P14: STATUS exits at correct count (not just valid transition)
always @(posedge clk)
    if (f_past_valid && rst_n && $past(ena) && $past(state) == ST_STATUS && state == ST_DONE) begin
        if ($past(rom_mode)) assert($past(status_cnt) == 4'd9);
        else                 assert($past(status_cnt) == 4'd3);
    end
```
This closes the gap between "valid state transitions" (P5) and "correct transition triggers." If someone changed the comparison from `==9` to `==8`, P5 would still pass but P14 would catch it.

### 2. 50th Cocotb Test: Full ROM Self-Index Sweep

Added `test_full_rom_self_index_sweep` to `test/test_stress.py`:
- Reads all 80 ROM entries via the serial protocol
- Verifies each record's self-index (byte 9 = rom_data[79:72]) matches its address
- Verifies no entry is all-zeros (all 80 should be populated)
- Provides simulation-level regression coverage for ROM integrity (complementing formal fv_rom.sv)

Test count: 5 anchor + 29 decoders + 16 stress = **50 tests**.
