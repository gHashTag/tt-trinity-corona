## 3. Scientific Research Findings (Loop 10)

### INT8 Quantization: Oracle Design Is Correct

Research confirms: INT8-to-float dequantization requires runtime scale and zero_point parameters (`float = scale * (int8 - zero_point)`). A conformance oracle without these parameters should output the raw sign-extended integer, not an FP32 value. This is what our decoder does.

### MXINT8 (OCP MX v1.0): Different From Plain INT8

OCP MX defines MXINT8 as **2's complement with implicit 2^-6 scale** (1 sign + 1 integer + 6 fractional bits). It also uses shared E8M0 block exponents per 32-element block. This is fundamentally different from our INT8 decoder (which handles generic signed integers). MXINT8 could be a future decoder addition.

### GF180MCU Timing: 50 MHz Is Conservative

Gate delays at 180nm: inverter ~0.16ns, NAND ~0.34ns, XOR ~1ns. A 20ns clock period accommodates 15-30 logic levels with wire delay. Our deepest combinational path (posit8 regime decode with barrel shifting) fits comfortably.

### TT Submission Path: LibreLane 3.0

LibreLane (successor to OpenLane2, FOSSi Foundation, v3.0) is the recommended tool. Submission: `tt/tt_tool.py --gf --harden`. Requires Python 3.11+, Docker, PDK=gf180mcuD. GitHub Actions (`tt-gds-action`) automates the full flow. TTGF26a deadline: 2026-06-22, chips expected ~2026-11-15.

### BF16 Exhaustive Test Performance

Research estimates: cocotb + Icarus Verilog at ~10k-50k iterations/sec for simple combinational decode. The 65,536-iteration BF16 exhaustive sweep should complete in **1-10 minutes**.
