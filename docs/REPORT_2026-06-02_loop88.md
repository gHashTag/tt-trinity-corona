## Why LNS8 (and the LUT specifically)

LNS8 encodes 1 sign + 7-bit Q3.4 logarithm; the decode is an antilog
`magnitude = round(256 * 2^(frac/16)) << int_part` (Q8.8). The 16 fractional LUT
entries (2^(0/16)..2^(15/16) scaled to Q0.8) are hand-written constants -- a
transcription/rounding error there would be invisible to `RTL == ref_model`
because a copy-paste of the same table sits in both. So the highest-value check
is the LUT constants against the closed-form `round(256*2^(i/16))`, plus the full
shift/zero/sign behaviour over all 256 codes against the actual RTL.

## Research (this loop, web available)

Logarithmic Number Systems represent a value as a sign plus the base-2 logarithm
of its magnitude, turning multiply/divide into add/subtract; the decode back to
linear is an antilog `2^x`. Corona's LNS8 stores the log as Q3.4: the integer
part selects the power-of-two octave (a shift) and the 4-bit fraction indexes a
table of `2^(frac/16)`. Splitting `2^(int+frac/16) = 2^int * 2^(frac/16)` into a
shift plus a small LUT is the standard hardware antilog technique (table +
barrel shift), so the only place for error is the rounded LUT -- which this test
pins to the closed form. Entry 8 is `2^(1/2) = sqrt(2) ~= 1.414` (362/256), a
convenient spot check.

Sources: [Logarithmic number system (Wikipedia)](https://en.wikipedia.org/wiki/Logarithmic_number_system),
[E. Swartzlander, "The Sign/Logarithm Number System" (IEEE TC, 1975)](https://doi.org/10.1109/T-C.1975.224311).
