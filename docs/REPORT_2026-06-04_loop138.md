# Loop 138 Report -- 2026-06-04

## Summary

**Turned the arc's recurring insight -- every defect found was "a field one bit too
narrow" (gf16_mul mantissa, bitnet neuron_base, gf_formats IDs) -- into a systematic
project-wide detector, ran it across all 279 RTL files in the four dies + the t27

dequantized with `fp16_out = {int4_signed[14:0], k'b0}` per scale_exp, but

dequantized incorrectly. Fixed it to a sign-extend + arithmetic shift
(`{{12{int4_signed[3]}}, int4_signed} <<< scale_exp`) -- correct for negatives, no
out-of-range select, still R-SI-1 (no `*`); also removed a dead `scale` LUT and an
undriven `quant` reg. Verified 256/256 (all 16 int4 x 16 scale_exp) vs reference on
all three dies; lint-clean. int4_quantizer is dead code (not instantiated), so no
silicon impact -- a library fix. After it, the sweep across 279 files shows only the
three intentionally-frozen `gf16_mul` silicon copies (whose fix lives in gf16_v2_mul
+ the t27 master). Saved the detector as `tools/width_defect_sweep.py`.**

## Why

A defect class that recurs is worth detecting systematically, not one unit at a time.
A from-top lint (loops 127-128) only sees instantiated logic; a per-file sweep also
catches standalone modules -- which is where int4_quantizer hid. Turning the insight
into a reusable, runnable detector is the durable form of the lesson.

## Implementation

- `tools/width_defect_sweep.py`: `verilator --lint-only -Wall` per RTL file across the
  five RTL dirs; reports `value too large for N bit` + `SELRANGE` / out-of-range
  selects. Exit nonzero if any remain.
- `int4_quantizer.v` (gamma fixed; ported to phi + euler): rewrote the dequant as
  sign-extend + `<<<`; removed the dead `scale`/`quant` regs. 256/256 vs reference,
  lint-clean, each committed (gamma ed91992, phi cbe6f2c, euler fbb9cf3).

## Research (this loop, web available)

The methodological payoff: once you notice a class ("field one bit too narrow"), the
right move is a cheap detector for the whole class, run everywhere, rather than
continuing to find instances by hand. The four instances this arc surfaced span a
register mantissa, a function argument, an enum field, and a bit-select slice -- all
the same underlying error, all caught by width-aware static analysis, and several
invisible to functional tests (dead code, or only triggered near a boundary). The
detector also encodes the negative result: 279 files, 0 remaining live defects (only
the 3 frozen-by-design copies), which bounds the search and is itself worth recording.
