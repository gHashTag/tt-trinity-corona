## Critical Finding: Missing SDC

No `.sdc` file existed anywhere in the project. LibreLane was using auto-generated constraints from CLOCK_PERIOD/CLOCK_PORT only -- no input/output delays, no max transition, no max fanout. This is the root cause of all GDS timing violations:

| Pre-Repair Violation | Count | OpenROAD RSZ Action |
|---------------------|-------|-------------------|
| Slew violations | 396 | Inserted 387 buffers |
| Fanout violations | 47 | Resized 362 instances |
| Setup violations | 20 | Repaired all 20 (100%) |
| Hold violations | 0 | None needed |

While OpenROAD repaired all violations, proper SDC constraints will reduce the repair burden and improve timing margin.

### New SDC: `src/constraint.sdc`

```tcl
create_clock -name clk -period 40.0 [get_ports clk]
set_input_delay  -clock clk -max 10.0 [get_ports {ui_in[*] uio_in[*] rst_n ena}]
set_input_delay  -clock clk -min  1.0 [get_ports {ui_in[*] uio_in[*] rst_n ena}]
set_output_delay -clock clk -max 10.0 [get_ports {uo_out[*] uio_out[*] uio_oe[*]}]
set_output_delay -clock clk -min  0.0 [get_ports {uo_out[*] uio_out[*] uio_oe[*]}]
set_max_transition 10.0 [current_design]
set_max_fanout 16 [current_design]
```

I/O delays set to 25% of clock period (10 ns) -- conservative for TT's RP2040 MUX interface.

## Metal5 PDN Fix Attempt

Research confirmed: **Metal5 is strictly forbidden** for user designs on TTGF26a. TT handles Metal5 power distribution at the top level. Our tile's PDN was generating Metal5 straps, causing precheck failure.

**Fix**: Added `FP_PDN_MULTILAYER: false` to config.json to suppress multi-layer PDN generation. The TT template DEF already provides power pin geometry. For our low-utilization design (2,308 cells, ~14% of 4x4 budget), stdcell Metal1 rails should suffice.
