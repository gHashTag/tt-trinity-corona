## Research (this loop, web available)

The issue fixed is *documentation drift between two records of the same intent* --
a known maintenance hazard that single-source-of-truth practice exists to prevent.
Two lightly-divergent question lists, each claiming to be the other, is exactly
the kind of stale cross-reference that erodes trust in project docs. The fix
follows the same discipline applied to code this whole arc: make each document
state precisely what it is, and link rather than duplicate. ADRs remain the
authoritative decision record; PLAN and OPEN_QUESTIONS now point to them rather
than re-asserting open status.

Sources: [Architecture Decision Records (adr.github.io)](https://adr.github.io/),
[single source of truth (docs drift)](https://dev.to/codinsonn/the-joy-of-single-sources-of-truth-277o).
