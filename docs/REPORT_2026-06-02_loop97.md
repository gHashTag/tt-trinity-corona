## Implementation

`test/test_mutation_guards.py` rewritten around a 17-entry `DECODERS` table
(width, instantiation, port-decl, reference, optional structural fallback):
- `_literal_variants(src)` yields the source with one `N'h/b/d` literal's LSB
  toggled (skipping x/z), masked to its width.
- `find_killing_mutation` sweeps the real module for a baseline; tries literal
  variants (capped at 50) until a compiled mutant differs from the reference on
  >=1 input; else applies the structural fallback; else reports NO-KILL.
- Bounded sweep (`SWEEP_LIMIT=512`) keeps the full 2^16/2^19 decoders fast while
  still observing a mutated output.
- References reused verbatim from the Loops 86-90 modules (float, posit8, lns8,
  simple, LUT) -- no re-implementation.

## Research (this loop, web available)

Mutation testing's effectiveness is measured by the mutation score (killed /
total non-equivalent mutants); a surviving mutant marks a test that cannot
distinguish correct from faulty behaviour. The well-known practical obstacles are
(a) equivalent mutants -- syntactic changes with no observable effect, which can
never be killed and must be excluded -- and (b) cost, since each mutant is a
separate build/run. This gate addresses both: it only accepts a literal mutation
once it has empirically CHANGED a swept output (so equivalent mutants are skipped
by construction, not assumed), and it bounds the sweep and stops at the first
killing mutation per decoder to keep the build/run cost to ~2s. The result is a
mutation score of 19/19 (100%) over the chosen, non-equivalent mutant set,
establishing that every decoder reference and both codegen guards are
discriminating.

Sources: [Mutation testing (Wikipedia)](https://en.wikipedia.org/wiki/Mutation_testing),
[Equivalent mutant problem (overview)](https://en.wikipedia.org/wiki/Mutation_testing#Equivalent_mutants).
