**Technique**: SymbiYosys BMC (Bounded Model Checking) with z3 at depth 1.
For combinational modules, this is an exhaustive proof over all possible
inputs — mathematically equivalent to testing every input, but verified by
an SMT solver rather than simulation.

**Tools**: `sby` (SymbiYosys) + `yosys-smtbmc` + `z3` (installed via
Homebrew). Each proof completes in <1 second.

## Research: Scientific Context

### No Prior Art for Silicon Format Oracles
Literature review confirms Corona is unique — no prior work combines:
1. Format metadata ROM on silicon (80 formats × 10 bytes)
2. On-die format converters (18 RTL decoders)
3. Deterministic spec-first code generation (.t27 → Verilog)

### Key References
- **FPnew/CVFPU** (ETH Zurich/OpenHW): closest open-source multi-format FPU,
  but computes (add/mul/div), doesn't characterize. Apache-2.0. GitHub:
  openhwgroup/cvfpu
- **NVIDIA Blackwell DE**: dedicated Decompression Engine for FP4/FP6/FP8
  format conversion at 50-200× speedup. Validates Corona's decode-only approach
- **Infineon formal FP verification** (arXiv:2512.06850): RTL-to-RTL formal
  verification of FP adder using SVA + SymbiYosys. Same toolchain we now use
- **Takum arithmetic** (Hunhold 2024, arXiv:2408.10594): 38% latency, 50%
  area reduction vs posit. Hardware codec exists in VHDL. Potential future
  decoder for Corona
- **OCP MX v1.0**: Microscaling spec confirmed. E8M0 scale factor, MXFP8
  block structure. Corona's ROM correctly documents the format
- **CalligoTech TUNGA**: first commercial posit RISC-V tapeout. Corona's
  posit8_decode would be among the first open-source posit decoders on GF180MCU

### TT08 Routing Lesson
TT08 demoscene projects found routing congestion (not cell count) is the
real area limiter. Corona's 80-entry ROM case statement creates a wide mux
tree that could stress routing. Budget 2× routing area as safety margin.
