## Research Findings

### IEEE P3109 (Minifloat Standard)
Still **unratified** as of June 2026. Notable: exponent bias vote was reversed in July 2025, meaning P3109 and OCP MX formats may diverge. No action needed for Corona — catalog uses OCP MX entries.

### OCP MX Format Ecosystem
No MX v2.0 spec. **NVFP4** (NVIDIA Blackwell-native FP4, block-16, E4M3 scale) emerged as a distinct production format, merged into llama.cpp. Strong candidate for catalog entry #81 if a ROM update is pursued post-tapeout.

### GF180MCU PDK Updates
- Fast-corner liberty files added (useful for multi-corner STA)
- New 1KB SRAM macro (unclear if TT-compatible)
- DRC false-positive fix (10nm rule)
- 25 MHz target confirmed as conservative for 180nm

### BitNet 1.58b in Production
Microsoft's 2B4T model (2B params, MIT license) and Bonsai 8B (~1GB, runs on iPhone via MLX) demonstrate real-world deployment. BitNet's `status_id` in the catalog could be upgraded from `[Spec]` to `[Empirical fit]`.

### FLoPS (Lean Formalization)
Published as Rutgers TR DCS-TR-762, Feb 2026. Found that ExtractScalar properties do NOT extend to 1-bit precision formats — relevant for any catalog entry with mantissa_bits=0.
