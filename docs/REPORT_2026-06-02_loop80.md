## The change

### New: `tools/ssot_layout.py` (single SSOT reader)
One module parses `rom_layout.t27` / `corona_oracle.t27`:
- `field_offsets()` -> `{NAME: (hi, lo, width)}` for the 11 record fields
- `const_enum(text, prefix)`, `cluster_counts()`, `scalar(name)`
- `extract(word, field_name)` -> pull a field from a packed 80-bit word using the SSOT-declared `(lo, width)`
- `LAYOUT` cached at import; `__main__` pretty-prints the layout (self-documenting)

### Refactor 1: `test_rom_spec_crosscheck.py`
`extract_fields()` no longer hardcodes `>> 56`, `>> 52`, ... -- it calls
`ssot_layout.extract(word, "TOTAL_BITS")` etc. The third literal copy of the
layout is gone. Output unchanged: **23/23 formats PASS vs published specs.**

### Refactor 2: `test_ssot_layout_crosscheck.py`
Dropped its private `parse_fields` / `parse_const_enum` / `parse_cluster_counts`
/ `_read` in favour of thin aliases onto `ssot_layout`. The cross-check now pins
`gen_rom.py`'s packer to whatever the ONE shared parser reads. Output unchanged:
**33/33 checks PASS.**

## Research (this loop, web available)

The single-source-of-truth register-description practice is exactly what this
refactor approximates in miniature:

| Source | Finding | Bearing on Corona |
| --- | --- | --- |
| SystemRDL (Accellera) | author registers ONCE; every "view" (RTL, UVM checker, C header, docs) is generated, so implementation and checker "can never drift out of sync" | The next step beyond this loop: generate `pack_record` itself from the `.t27`, not just the extractor's offsets |
| PeakRDL / systemrdl-compiler | open-source Python toolchain: SystemRDL -> RTL + UVM + C + HTML | A realistic vehicle for a Corona `.t27` -> packer generator (Loop 79 Option C) |
| Semifore / Agnisys / Arteris Magillem | commercial CSR compilers; "1000+ error checks" from one golden source; IP-XACT is an *exchange* format, not an authoring one | Confirms `.t27` (authoring) + generated consumers is the right shape; IP-XACT only if external interchange is needed |

Takeaway: reducing 3 hand-copies to 1-reader + generated-consumers is the
correct direction; the asymptote is generating the packer too.

## Three Collaboration Options for the Next Loop

### Option A: Generate the packer from the SSOT (close the loop to 1 copy)
Emit `pack_record` (or its shift table) from `tools/ssot_layout.field_offsets()`
so the packer is derived, not hand-written. Then the layout has exactly ONE
source and the "mechanical SSOT->RTL" claim is literally true at the bit level.
(Per SystemRDL / PeakRDL practice.)

### Option B: Formal ROM miter (carried from Loop 79 A)
Finite ROM (80x80 bits). SymbiYosys/EQY proof that `format_rom.v` == the packed
CATALOG for all 80 addresses -- a closed-form equivalence replacing sampled
checks. Needs `sby`/`yosys` in CI (yosys already used by the GDS/formal jobs).

### Option C: Property-based test of the parser/packer round-trip
Use `hypothesis` to generate random field tuples, pack via `gen_rom`, extract
via `ssot_layout`, and assert round-trip identity + that no field bleeds into a
neighbour. Hardens the ONE remaining parser against edge cases beyond the fixed
probe vector.
