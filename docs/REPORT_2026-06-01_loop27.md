## Implemented: k-Induction Unbounded Proof

### Problem

BMC (bounded model checking) at depth 30 proves properties for up to 30 clock
cycles. k-Induction (`mode prove`) provides an **unbounded** mathematical
guarantee — no assertion violation at ANY depth, for all time.

The induction step failed because the solver picked unreachable register
combinations (e.g., timer=18 with data_cnt=15 in DATA state) that satisfy all
assertions but don't correspond to any reachable state. Additionally, a 5-bit
arithmetic overflow in the timer+data_cnt invariant allowed the solver to
bypass the constraint (18+15=33 mod 32=1, which is ≤ 16).

### Solution: 7 Auxiliary Invariants

Added 7 invariants relating `f_idle_timer` to FSM state and counters. Each
was hand-proven correct before implementation by tracing all state transitions.

| # | Invariant | What it constrains |
|---|-----------|-------------------|
| AI1 | `CMD2 → timer == 0` | Timer resets in IDLE, CMD2 sees 0 |
| AI2 | `DATA → timer + data_cnt ≤ 16` | Conserved quantity from CMD2 entry (6-bit math) |
| AI3a | `STATUS ∧ !rom → timer ≤ sc + 16` | Decode: entry timer ≤ 16, tracks sc |
| AI3b | `STATUS ∧ rom → timer == sc + 1` | ROM: entry timer = 1, lockstep with sc |
| AI4 | `DONE → timer ≤ 20` | Max path: 15 DATA + 4 STATUS + 1 CMD2 |
| AI5 | `CMD2 → status_cnt == 0` | Set in IDLE, unchanged in CMD2 |
| AI6 | `CMD2 → rom_mode == 0` | Set in IDLE, unchanged in CMD2 |
| AI7 | `STATUS → sc ≤ 9 (rom) / 3 (decode)` | Tight bounds for induction step |

**Key insight**: AI2 uses 6-bit arithmetic (`{1'b0, timer} + {2'b00, data_cnt}`)
to prevent overflow. The original 5-bit version wrapped at 32, allowing the
solver to exploit states where timer=18, data_cnt=15 (sum=33 mod 32=1 ≤ 16).
