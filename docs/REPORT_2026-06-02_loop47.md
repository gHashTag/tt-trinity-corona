## Three Collaboration Options for Next Loop

### Option A: Push to GitHub + Trigger GDS Hardening
57 commits unpushed. **This is the only remaining blocker for tapeout submission.** With all verification passing (76 GLS + 80 ROM golden + 19 formal + 27 cocotb), the design is ready for hardening. Requires `git push` with GitHub PAT. STA at 25 MHz will reveal actual timing margins on GF180MCU.

### Option B: Full 10-Byte ROM Readback Validation
Current ROM golden test checks 4 of 10 bytes per record (phi_distance, ref_index, flags). Extend to all 10 bytes (add format_index_id, cluster_id, status_id, total_bits, sign_bits, exp_bits, mant_bits, encoding_kind). This would verify 800 bytes total — complete bit-perfect ROM content through synthesis.

### Option C: D2D Stub Implementation
Implement a minimal D2D passthrough on `uio[3:0]` (TX) / `uio[7:4]` (RX) that echoes format queries to the companion Gamma die. Even as a stub (no actual Gamma model), this ensures the D2D pins are properly driven and not floating on the TT carrier. ~50-100 cells overhead.
