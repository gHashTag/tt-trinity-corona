## Why

A staged fix nobody checks rots; the respin brief claims "verified fix staged on
every affected die", so the fixes should actually be re-verified in CI on those
dies, not just in Gamma. This makes the brief's claim self-enforcing and means a
future respin can trust the staged modules without re-deriving the verification.

## Research (this loop, web available)

The point is verification durability: an audit's value decays unless its checks are
executed continuously, and a fix that is staged but ungated is indistinguishable from
a fix that has silently regressed. Putting the same gate on every die that shares the
defect (and the fix) mirrors how the defect propagated -- one code-gen fanned the bug
to three dies, so the verification has to be fanned to all three too, or the brief's
"verified everywhere" is aspirational. This is the CI corollary of the
single-source-of-truth discipline that recurs through this project's history.
