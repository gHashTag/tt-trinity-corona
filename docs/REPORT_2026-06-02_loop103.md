## Why

The dossier is human-readable markdown; a preprint evidence table, a dashboard, or
any downstream tooling wants the same data as JSON. Emitting it from the same
generator (single source) gives a stable, parseable artifact (`schema`,
per-decoder rows, summary counts, CI-gate list) that is kept honest by the same
freshness gate as the markdown -- consistent with the ROM / bring-up-vector /
dossier generation pattern.

## Research (this loop, web available)

Machine-readable, schema-tagged evidence is the basis of automated assurance:
structured assurance-case exchange formats (e.g. OMG SACM -- Structured Assurance
Case Metamodel) exist precisely so that a correctness argument and its evidence
can be consumed by tools, not just read. `verification.json` is a lightweight
instance of that idea -- a single generator emits both the human view (markdown)
and the machine view (JSON) from one data model, and a regenerate-and-diff gate
keeps both faithful to the actual test suite.

Sources: [OMG SACM -- Structured Assurance Case Metamodel](https://www.omg.org/spec/SACM/),
[generated-file CI gate (git-diff)](https://git-scm.com/docs/git-diff).
