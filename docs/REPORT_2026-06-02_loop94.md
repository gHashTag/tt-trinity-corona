## Research (this loop, web available)

The not-implemented contract follows the established practice that hardware
should emit a STRUCTURED, self-describing response for unimplemented/illegal
operations rather than undefined behaviour. Historically, transistor-budget-
limited CPUs (6502, 8086, Z80) omitted invalid-opcode detection, yielding
"illegal opcodes" with undocumented effects; modern designs trap or return a
defined code. Corona returns `[0xFF, fmt_id, status_id, 'N']` -- an explicit
"format not on-die" signal carrying the queried fmt_id and its claim-status,
which a host can parse. This loop pins those constants to the RTL and to the
gen_rom status enum so the bring-up oracle and the silicon agree on the contract.

Sources: [Illegal opcode (Wikipedia)](https://en.wikipedia.org/wiki/Illegal_opcode),
[HW support for emulating unimplemented instructions (USPTO 5,632,028)](https://image-ppubs.uspto.gov/dirsearch-public/print/downloadPdf/5632028),
[Symbolic Quick Error Detection (arXiv:2006.05449)](https://arxiv.org/pdf/2006.05449).
