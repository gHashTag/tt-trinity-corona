## Research (this loop, web available)

Post-silicon Fmax characterization via shmoo / frequency sweep is the standard
way to find a chip's maximum operating frequency and its speed-limiting path: run
known-good vectors while ramping the clock and locate the first failure. For a
single-clock-domain combinational oracle, the first decoder to fail as frequency
rises *is* the critical path, so the sweep doubles as critical-path
identification -- no on-chip timing instrumentation needed. The design here keeps
the measurement logic separate from (and testable without) the board, then wires
to hardware via a single clock-set callback at bring-up.

Sources: [Shmoo plot (frequency/voltage characterization)](https://en.wikipedia.org/wiki/Shmoo_plot),
[post-silicon validation -- speed binning / Fmax](https://en.wikipedia.org/wiki/Post-silicon_validation).
