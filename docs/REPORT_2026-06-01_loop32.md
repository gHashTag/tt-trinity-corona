## Fixes Applied

### 1. byte_count Clamp (Protocol Hardening)

**Problem**: The CMD2 state accepted `ui_in[3:0]` (0-15) as `data_cnt`, but the shift register is 32 bits (4 bytes). If host sends byte_count=5..15, the FSM accepts all bytes but only the last 4 are retained — silent data loss.

**Fix**: Clamp data_cnt to 4 in CMD2:
```verilog
data_cnt <= (ui_in[3:0] > 4'd4) ? 4'd4 : ui_in[3:0];
```

**Impact**: No current format uses > 3 bytes, so no functional change for valid inputs. But a malformed command no longer silently loses data — the FSM clamps and processes the last 4 bytes deterministically.

### 2. Tightened Formal Bounds

With byte_count clamped to 4, the worst-case transaction path changes:

| Path | Old | New |
|------|-----|-----|
| Decode (max) | 1+15+4+1 = 21 cycles | 1+4+4+1 = 10 cycles |
| ROM readback | 1+0+10+1 = 12 cycles | 1+0+10+1 = 12 cycles (unchanged) |

Updated properties:
- **P10**: Liveness bound 21 → 12
- **P13** (new): `data_cnt <= 4` in DATA state
- **AI2**: Timer + data_cnt ≤ 16 → ≤ 5
- **AI3** (decode): Timer ≤ status_cnt + 16 → ≤ status_cnt + 5
- **AI4**: Timer in DONE ≤ 20 → ≤ 11
- **Cover C3**: data_cnt == 15 → data_cnt == 4 (15 no longer reachable)

## Research Findings

### Posit(8,0) vs Posit(8,2)
The 2022 Posit Standard fixes es=2 for all precisions, including 8-bit. Our decoder implements Posit(8,0) (pre-standard, widely used in ML). Both are valid choices but the difference should be documented. The ROM record for Posit8 should specify "es=0" explicitly.

### NF4 Values Confirmed
The QLoRA quantile table has no known errata. A 2023 paper critiques theoretical optimality but confirms empirical values. Our golden reference matches exactly.

### LNS8 Not Standardized
No industry standard exists for 8-bit LNS. Our Q3.4 format with round-to-nearest LUT is mathematically sound. Recent papers (DATE 2025, NeurIPS 2024) explore different bit allocations (4+3 vs 3+4) — our choice should be labeled experimental.

### MXFP8 / E8M0 Confirmed
OCP MX v1.0 spec confirms our E4M3 decode rules (bias=7, NaN only at 0x7F/0xFF). Our MXINT8 -128→NaN convention is project-specific, not spec-mandated.

### Corona is First TT Project with Formal CI
No published Tiny Tapeout project uses formal verification in its CI pipeline. This is a distinguishing strength.
