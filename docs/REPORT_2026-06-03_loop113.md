## Why

Every verification gate this arc added required a matching hand-written CI step --
done 20 times, an obvious maintenance smell and an easy thing to forget. A single
auto-discovering runner removes that per-gate ceremony and makes the gate set
self-extending: drop a new `test/test_*.py` with a `main()` and it runs in CI
automatically.

## Implementation

- `tools/run_verification_gates.py`: discover (glob `test/test_*.py`, skip files

## Research (this loop, web available)

Test discovery -- collecting and running all tests matching a convention rather
than enumerating them -- is the norm in mature frameworks (pytest, unittest
discover) precisely to avoid a hand-maintained registry that drifts from the
actual test files. Applying it to the CI gate list removes the per-gate step and
the "forgot to wire it up" failure mode; the explicit floor preserves the one
safety property a bare glob loses (catching a discovery bug that silently runs
fewer tests). This is the same self-extending, drift-resistant pattern used for
the generated artifacts elsewhere in the repo.

Sources: [pytest test discovery](https://docs.pytest.org/en/stable/explanation/goodpractices.html#test-discovery),
[unittest discovery](https://docs.python.org/3/library/unittest.html#test-discovery).
