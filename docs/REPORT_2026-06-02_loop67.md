| Single clock domain | Only `posedge clk` in entire RTL. All decoders are purely combinational (`always @(*)`). Format ROM is synchronous on `clk`. No generated clocks, clock gating, or divided clocks. No CDC analysis needed. |
| Anchor value collision | Anchor `16'h47C0` outputs on `{uio_out, uo_out}` only when `fmt_id=0x7F`. During decode, `uio_out=0x00` always (uio_oe=0). No decoder can produce `uo_out=0xC0` combined with `uio_out=0x47` — the 16-bit combination is architecturally exclusive to anchor mode. |

## Scientific Research

### New Numeric Formats Discovered

- **HiFloat8 (HiF8)** (Huawei, Feb 2026, arXiv:2602.12635): Tapered-precision 8-bit format that dynamically trades mantissa/exponent bits by magnitude. 0.3-0.5% accuracy advantage over static FP8. Hardware-deployed on Ascend NPUs. Novel decoder challenge — distinct from fixed-field formats in Corona's catalog.
- **HiFloat4 (HiF4)** (Huawei, Apr 2026, arXiv:2604.08826): Three-level hierarchical scaling (E6M2 block / E1 sub-block / E1 micro-block) with S1P2 element values. Pre-training results on Ascend. Architecturally distinct from MXFP4.
- **NVFP4** (NVIDIA): 16-element blocks (not 32) with E4M3 scaling factor (not E8M0 power-of-two). Distinct from MXFP4.
- **Action**: These are post-submission catalog additions (formats #81+). Current 80-format ROM is fixed for TTGF26a.
