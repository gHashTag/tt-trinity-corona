## Research (this loop, web available)

A datasheet's timing section conventionally separates the deterministic, digital
part (transaction latency in cycles -- known pre-silicon for a fixed protocol)
from the analog maximum frequency (Fmax -- measured post-silicon). Publishing the
cycle latency now with Fmax explicitly marked TBD is standard practice: it states
the contract the silicon must meet and reserves the measured number. The fix to
the module count is the same single-source hygiene applied across the docs -- the
datasheet must agree with the README, PLAN, and the on-die set rather than carry
an independent (stale) figure.

Sources: [Datasheet timing parameters (latency vs max frequency)](https://en.wikipedia.org/wiki/Datasheet),
[Shmoo plot / Fmax post-silicon characterization](https://en.wikipedia.org/wiki/Shmoo_plot).
