## Research (this loop, web available)

The format-ID bug is the classic enum-outgrows-its-field defect: an identifier space
quietly grew past 16 entries while the constants kept a 4-bit literal, so the new
IDs wrapped onto old ones -- invisible because nothing dispatches on them yet, and
invisible to functional tests for the same reason. Width-aware static analysis is the
only thing that flags a dormant collision like this; it is the same fault family as
the gf16_mul mantissa width, one level up (a register vs. an enum). The lesson for a
shared master is to lint the whole set, not just the unit you came to fix -- the
high-value defects this arc found (gf16_mul, bitnet neuron_base, gf_formats IDs) are
all "a field one bit too narrow", and a single -Wall pass surfaces the whole class.
