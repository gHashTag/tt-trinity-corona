## The gap

Loops 86-90 reached 17/17 decoders pinned to external references, but nothing
guaranteed it stayed that way. This is the classic "meta-test / coverage ratchet"
need: enforce that every module has a test, globally and automatically, so a new
module is covered by default rather than relying on a developer remembering to
add the oracle. Without it, 17/17 was a one-time measurement, not a property.

## Research (this loop, web available)

This is the **ratchet / meta-test** pattern: a CI check that only allows
one-directional improvement and fails when an undesirable count (here, decoders
lacking an independent oracle) rises. The literature's key recommendations all
apply: enforce per-module rather than aggregate (a new untested module must fail
immediately, not be averaged away); apply the enforcement globally/automatically
so new modules are covered by default; and prefer a meta-test that compares the
covered set to the actual module set. The gate implements exactly this -- the
"undesirable count" is `decoders - covered`, required to be empty.

Sources: [Ratchets: improving systems incrementally (Burwell)](https://www.dustyburwell.com/2019/05/29/ratchets),
[Ratchets in software development (qntm)](https://qntm.org/ratchet),
[How to ensure test coverage (Testim)](https://www.testio.io/blog/how-to-ensure-test-coverage/).
