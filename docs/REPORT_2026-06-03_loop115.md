## Research (this loop, web available)

Two patterns: (1) derive-don't-enumerate -- the generator now computes the gate
list from the filesystem the same way the runner does, eliminating the duplicate
registry that had silently drifted (the recurring single-source lesson of this
arc); (2) shields.io endpoint badges -- a generated JSON (`schemaVersion`, `label`,
`message`, `color`) consumed by `img.shields.io/endpoint`, the standard way to put
a self-hosted, always-current status badge in a README without a third-party
service computing it. Both keep the public-facing numbers tied to the actual
artifact rather than a manually-updated copy.

Sources: [shields.io endpoint badge](https://shields.io/badges/endpoint-badge),
[pytest/unittest test discovery](https://docs.pytest.org/en/stable/explanation/goodpractices.html#test-discovery).
