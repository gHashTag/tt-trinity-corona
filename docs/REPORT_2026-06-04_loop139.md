## Research (this loop, web available)

The arc's closing discipline: detect -> fix -> gate. A class of bug that recurred
four times (and was invisible to functional tests because the instances were dead
code or only fired near a boundary) is exactly what a cheap static gate is for -- it
turns "we cleaned it up" into "it cannot come back". The allow-list is the honest
part: rather than suppress the gate globally or edit frozen silicon to silence it, it
records the one expected exception with a reason, so the gate stays meaningful (any
new SELRANGE/overflow elsewhere fails CI) while respecting the taped-out source.
This is the standard "ratchet" pattern -- lock in a cleaned invariant so regressions
are caught at the door, not in the next silicon audit.
